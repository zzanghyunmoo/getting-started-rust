## 1. 컴퓨터 과학이란?

### 1.1 정의

컴퓨터 과학(Computer Science)은 **컴퓨터와 계산(computation)에 관한 학문**입니다. 단순히 컴퓨터를 사용하는 법을 배우는 것이 아니라, **"어떻게 효율적으로 문제를 해결할 것인가?"**를 연구하는 학문입니다.

### 1.2 핵심 연구 분야

| 분야 | 설명 | 예시 |
|------|------|------|
| **알고리즘** | 문제 해결 절차 | 정렬, 탐색, 최단 경로 |
| **자료구조** | 데이터의 조직화 방법 | 배열, 리스트, 트리, 그래프 |
| **프로그래밍 언어** | 인간과 컴퓨터의 소통 도구 | Rust, Python, C++ |
| **컴퓨터 구조** | 하드웨어의 동작 원리 | CPU, 메모리, 저장장치 |
| **운영체제** | 시스템 자원 관리 | Linux, Windows, macOS |

### 1.3 프로그래밍의 본질

프로그래밍은 **"컴퓨터에게 일을 시키는 방법"**입니다.

```
문제 정의 → 해결책 설계 → 코드로 구현 → 테스트 → 유지보수
```

**프로그래밍의 핵심 요소:**
- **추상화(Abstraction)**: 복잡한 것을 단순하게 만들기
- **자동화(Automation)**: 반복 작업을 자동으로 처리
- **알고리즘적 사고**: 단계별 문제 해결 능력

## 2. 프로그래밍 언어의 역사와 분류

### 2.1 프로그래밍 언어의 진화

```
기계어 → 어셈블리어 → 고급 언어 → 현대 언어
(0/1)    (니모닉)      (C, Fortran)  (Rust, Python)
```

| 세대 | 언어 | 특징 |
|------|------|------|
| 1세대 | 기계어 | 0과 1로만 구성 |
| 2세대 | 어셈블리어 | ADD, MOV 같은 니모닉 사용 |
| 3세대 | 고급 언어 | C, Fortran, Java |
| 4세대 | 선언형 언어 | SQL, Prolog |
| 5세대 | 현대 언어 | Rust, Python, Go |

### 2.2 실행 방식에 따른 분류

**컴파일 언어** (Rust 포함):
```
소스 코드 → 컴파일러 → 기계어 → 실행
```
- 장점: 실행 속도가 빠름
- 단점: 컴파일 시간이 걸림

**인터프리터 언어**:
```
소스 코드 → 인터프리터 → 한 줄씩 실행
```
- 예시: Python, JavaScript
- 장점: 즉시 실행 가능
- 단점: 실행 속도가 상대적으로 느림

### 2.3 타입 시스템에 따른 분류

| 특성 | 정적 타이핑 | 동적 타이핑 |
|------|------------|-------------|
| 타입 검사 시점 | 컴파일 타임 | 런타임 |
| 예시 | Rust, C++, Java | Python, JavaScript |
| 장점 | early error detection | 유연함, 빠른 개발 |

**Rust는 정적 타이핑 언어**로, 컴파일 타임에 대부분의 오류를 잡아냅니다.

### 2.4 메모리 관리에 따른 분류

**Rust의 소유권(Ownership) 시스템:**
- 컴파일 타임에 메모리 안전성 보장
- 가비지 컬렉션 오버헤드 없음
- 수동 메모리 관리의 위험성 제거

## 3. Rust 언어 소개

### 3.1 Rust의 역사

```

2010: 오픈 소스로 공개
2015: Rust 1.0 정식 출시
2024: Rust 2024 Edition 출시
```

### 3.2 Rust가 해결하는 문제

Rust는 **C/C++의 메모리 안전성 문제**를 해결하면서도 **고성능**을 유지합니다.

| 문제 | C/C++ | Rust |
|------|-------|------|
| 댕글링 포인터 | 런타임 오류 | 컴파일 타임 감지 |
| 데이터 레이스 | 런타임 오류 | 컴파일 타임 감지 |
| 메모리 누수 | 수동 관리 | 자동 관리 |
| 성능 | 최고 | 최대한 가깝게 |

### 3.3 Rust의 핵심 특징

| 특징 | 설명 |
|------|------|
| **메모리 안전성** | 컴파일 타임에 보장 |
| **제로 비용 추상화** | 고수준 기능도 저수준만큼 빠름 |
| **무공포 동시성** | 데이터 레이스를 컴파일 타임에 감지 |
| **현대적 문법** | 패턴 매칭, 타입 추론 등 |
| **훌륭한 도구** | Cargo, rustfmt, clippy |

## 4. 컴파일러와 인터프리터

### 4.1 왜 컴파일러가 필요할까요?

컴퓨터는 **0과 1**만 이해할 수 있습니다. 하지만 우리는 **영어와 한국어**로 코드를 작성합니다. 이 둘 사이를 연결해주는 것이 바로 **컴파일러**입니다.

```
우리가 작성한 코드
    ↓  (번역이 필요함)
컴퓨터가 실행하는 기계어
```

**간단한 비유:**
- 소스 코드 = 영어로 된 요리 레시피
- 컴파일러 = 번역기
- 기계어 = 요리사(컴퓨터)가 이해할 수 있는 언어

### 4.2 컴파일러의 작동 원리 - 단계별 상세 설명

컴파일러는 소스 코드를 기계어로 변환할 때 여러 단계를 거칩니다. 각 단계가 왜 필요한지 함께 알아보겠습니다.

```
┌─────────────────────────────────────────────────────────────────────┐
│                         컴파일 과정 상세도                          │
└─────────────────────────────────────────────────────────────────────┘

    [1단계] 소스 코드 작성
    ─────────────────────────────────────────────────────────────────
    fn main() {
        println!("Hello, world!");
    }

                    ↓ 이 코드를 컴퓨터가 이해하려면?


    [2단계] 어휘 분석 (Lexical Analysis)
    ─────────────────────────────────────────────────────────────────
    "fn main() { println!(\"Hello\"); }"
              ↓
    토큰(Token)으로 분해
    ┌─────────┬────────┬────────┬──────────┬──────────┬────────┬──────┐
    │ Keyword │ Ident  │ Symbol │ Keyword  │ String   │ Symbol │ ...  │
    │   fn    │  main  │   (    │ println  │ "Hello"  │   !    │      │
    └─────────┴────────┴────────┴──────────┴──────────┴────────┴──────┘

    ※ 왜? 컴퓨터가 "단어" 단위로 인식하도록 하기 위해


    [3단계] 구문 분석 (Syntax Analysis)
    ─────────────────────────────────────────────────────────────────
    토큰들을 조립해서 "구조"를 만듭니다

    ┌─────────────────────────────────────┐
    │         AST (추상 구문 트리)         │
    ├─────────────────────────────────────┤
    │                                     │
    │            FnDef "main"             │
    │            /        \               │
    │        (params)    Block            │
    │                      │              │
    │                  MacroCall          │
    │                  "println!"         │
    └─────────────────────────────────────┘

    ※ 왜? 코드의 "문법"이 올바른지 확인하고 구조를 파악하기 위해

    예시: "나는 밥을 먹었다" vs "먹었다 밥을 나는" (문법 체크)


    [4단계] 의미 분석 (Semantic Analysis)
    ─────────────────────────────────────────────────────────────────
    변수의 타입, 스코프 등을 검사합니다

    ❌ 문법은 맞지만 의미가 틀린 경우:
       let x: i32 = "안녕";  // 타입 불일치!
       println!(y);          // y가 선언되지 않음!

    ※ 왜? "문법은 맞지만 의미가 맞는지" 확인하기 위해


    [5단계] 중간 코드 생성 (Intermediate Representation)
    ─────────────────────────────────────────────────────────────────
    특정 CPU에 종속되지 않는 중간 언어로 변환

    println!("Hello")  →  IR 코드
                         ┌────────────────────┐
                         │ %1 = call @println │
                         │     "Hello"        │
                         └────────────────────┘

    ※ 왜? 다양한 CPU(인텔, ARM 등)를 지원하기 위해 중간 단계가 필요


    [6단계] 최적화 (Optimization)
    ─────────────────────────────────────────────────────────────────
    불필요한 코드 제거, 더 빠른 코드로 변환

    [최적화 전]                  [최적화 후]
    let x = 5;              →    (제거됨 - 사용 안 함)
    let y = 10;                  let y = 10;
    println!("{}", y);            println!("{}", y);

    let result = 2 * 3;     →    let result = 6;
                              (컴파일 타임에 계산)

    ※ 왜? 프로그램을 더 빠르게 실행하기 위해


    [7단계] 코드 생성 (Code Generation)
    ─────────────────────────────────────────────────────────────────
    마침내 기계어로 변환!

    소스 코드      기계어 (실제 컴퓨터가 실행하는 코드)
    ──────────    ────────────────────────────────────────────────
    fn main() {    55 48 89 e5 48 83 ec 10 48 8d 3d 00 00 00 00 ...
        println!("Hello");
    }

    ※ 이 0과 1의 조합이 CPU에 명령을 내립니다
```

### 4.3 Rust 컴파일 파이프라인

Rust 컴파일러(rustc)는 위 과정을 더 구체적으로 구현합니다:

```
┌─────────────────────────────────────────────────────────────┐
│                    Rust 컴파일 과정                         │
└─────────────────────────────────────────────────────────────┘

source.rs (우리가 작성한 코드)
    │
    │ "fn main() { println!("Hi"); }"
    ↓
═══════════════════════════════════════════════════════════════
                    rustc가 시작됩니다!
═══════════════════════════════════════════════════════════════
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 1. Parser (구문 분석)                                        │
│    소스 코드 → AST (추상 구문 트리)                          │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. HIR lowering (고수준 IR 생성)                             │
│    AST → HIR (High-level Intermediate Representation)        │
│    - 타입 추론 완료                                          │
│    - 매크로 확장 완료                                        │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. MIR lowering (중수준 IR 생성)                             │
│    HIR → MIR (Mid-level Intermediate Representation)         │
│    - 제어 흐름 그래프 생성                                   │
│    - Borrow checking (빌림 검사) ★Rust 핵심★               │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. LLVM IR 생성                                              │
│    MIR → LLVM IR                                             │
│    - LLVM이 이해할 수 있는 형태로 변환                       │
└─────────────────────────────────────────────────────────────┘
    ↓
═══════════════════════════════════════════════════════════════
                 LLVM이 이어받습니다!
═══════════════════════════════════════════════════════════════
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. LLVM 최적화                                               │
│    - 인라이닝 (함수 호출을 코드로 대체)                      │
│    - 루프 최적화                                             │
│    - 불필요한 코드 제거 (Dead Code Elimination)             │
└─────────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────────┐
│ 6. 기계어 생성                                               │
│    LLVM IR → 기계어 (x86, ARM, etc.)                        │
│    - 운영체제가 실행할 수 있는 실행 파일 생성               │
└─────────────────────────────────────────────────────────────┘
    ↓
executable (실행 파일!)
```

### 4.4 실제로 확인해보기

Rust 컴파일러가 각 단계에서 무엇을 하는지 확인할 수 있습니다:

```bash
# 컴파일 단계별 출력 보기
cargo rustc -- --emit=asm  # 어셈블리 코드 출력
cargo rustc -- --emit=llvm-ir  # LLVM IR 출력
cargo rustc -- --emit=mir  # MIR 출력
```

### 4.5 컴파일러 vs 인터프리터 비교

| 특성 | 컴파일러 (Rust) | 인터프리터 (Python) |
|------|----------------|---------------------|
| **번역 방식** | 전체를 한 번에 번역 | 한 줄씩 번역 후 실행 |
| **번역 시점** | 실행 전 | 실행 중 |
| **실행 속도** | 빠름 (이미 번역됨) | 상대적으로 느림 |
| **오류 발견** | 컴파일 타임 (실행 전) | 런타임 (실행 중) |
| **배포** | 실행 파일만 배포 | 소스 코드 배포 |
| **개발 속도** | 컴파일 시간 필요 | 즉시 실행 가능 |

**비유:**
- **컴파일러**: 영화를 미리 자막 번역 → 상영时 바로 볼 수 있음
- **인터프리터**: 동시통역 → 말할 때마다 즉시 번역

### 4.6 LLVM과 Rust

**LLVM(Low Level Virtual Machine)**은 현대적인 컴파일러들의 **공통 인프라**입니다.

```
┌──────────────────────────────────────────────────────────────┐
│                   LLVM을 사용하는 언어들                      │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│   C/C++      Rust       Swift      Kotlin      Go            │
│    │          │          │          │          │             │
│    └──────────┴──────────┴──────────┴──────────┘             │
│                    │                                         │
│              ┌─────▼─────┐                                   │
│              │   LLVM    │  ← 모든 언어가 여기서 모임          │
│              └─────┬─────┘                                   │
│                    │                                         │
│              ┌─────▼─────┐                                   │
│              │  기계어   │  ← 한 번만 개발하면 됨!             │
│              │ (x86/ARM) │                                   │
│              └───────────┘                                   │
└──────────────────────────────────────────────────────────────┘
```

**LLVM을 사용하는 이유:**
1. 각 언어 개발자가 "모든 CPU에 대한 최적화"를 직접 만들 필요 없음
2. LLVM은 이미 최적화가 잘 되어 있음
3. 새로운 CPU가 나와도 LLVM만 업데이트하면 모든 언어가 혜택

### 4.7 왜 컴파일하는데 시간이 걸릴까요?

Rust 컴파일이 느린 이유는 **더 엄격한 검사**를 하기 때문입니다:

```rust
// Rust 컴파일러가 확인하는 것들:
1. 문법이 맞나요?              → 다른 언어도 함
2. 타입이 맞나요?              → 다른 언어도 함
3. 변수가 초기화되었나요?       → Rust 특징
4. 메모리 안전성이 보장되나?    → Rust 특징 ★
5. 데이터 레이스 가능성이?      → Rust 특징 ★
6. 스레드 안전성은?             → Rust 특징 ★
```

**더 엄격하게 검사할수록, 실행 시간에 오류가 날 확률이 줄어듭니다!**

rustup은 Rust 툴체인 설치 관리자입니다.

**Linux/macOS:**

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

**Windows:**
- https://rustup.rs/ 에서 설치 파일 다운로드

### 5.2 설치 확인

```bash
# 버전 확인
rustc --version
cargo --version

# 업데이트
rustup update
```

예상 출력:

### 5.3 예제 프로젝트 생성

먼저 간단한 예제 프로젝트를 생성하여 Rust의 기본을 익혀보겠습니다.

```bash
# 프로젝트 루트로 이동
cd /path/to/getting-started-rust

# 예제 프로젝트를 위한 디렉토리 생성
mkdir -p examples
cd examples

# Chapter 1 예제 프로젝트 생성
cargo new ch01
cd ch01

# 프로젝트 구조 확인
ls -la
```

생성된 구조:
```
examples/
└── ch01/
    ├── Cargo.toml          # 프로젝트 설정
    └── src/
        └── main.rs         # 메인 소스 코드
```

**프로젝트 구조 설명:**
- `Cargo.toml`: 프로젝트 이름, 버전, 의존성 등을 관리하는 설정 파일
- `src/main.rs`: 프로그램의 진입점이 되는 메인 소스 코드
- `src/`: 소스 코드가 위치하는 디렉토리

### 5.3.1 Cargo.toml 기본 구조

생성된 [Cargo.toml](Cargo.toml)을 확인해보세요:

```toml
[package]
name = "ch01"
version = "0.1.0"
edition = "2021"

[dependencies]
```

**기본 필드 설명:**
| 필드 | 설명 | 예시 |
|------|------|------|
| `name` | 프로젝트/패키지 이름 | `"ch01"` |
| `version` | 버전 정보 | `"0.1.0"` (Semantic Versioning) |
| `edition` | Rust 에디션 | `"2021"` (최신 에디션) |
| `[dependencies]` | 외부 라이브러리 의존성 | 필요한 crate를 추가 |

**참고**: 에디션(Edition)은 Rust의 주요 업데이트로, 2015, 2018, 2021, 2024 등이 있습니다. 새 프로젝트는 최신 에디션을 사용하는 것이 좋습니다.

### 5.3.2 여러 예제 파일 추가하기 (선택사항)

본 문서의 예제 코드들을 따라하다 보면 여러 개의 예제 파일을 만들게 됩니다. 이때 각 예제를 독립적으로 실행하고 싶다면 `[[bin]]` 설정을 추가할 수 있습니다.

**참고**: 현재 [examples/ch01/](../examples/ch01/) 프로젝트에는 이미 모든 예제 파일이 준비되어 있습니다. 따라서 필요한 예제를 `--bin` 옵션으로 실행할 수 있습니다:

```bash
# 특정 예제 실행
cargo run --bin 01_hello_world
cargo run --bin 04_string_interpolation
cargo run --bin 09_game_title_screen
```

**직접 여러 파일을 추가하고 싶다면:**

[Cargo.toml](Cargo.toml)에 `[[bin]]` 섹션을 추가하세요:

```toml
[package]
name = "ch01"
version = "0.1.0"
edition = "2021"

[dependencies]

# 여러 예제 파일을 바이너리로 등록
[[bin]]
name = "hello"
path = "src/hello.rs"

[[bin]]
name = "greeting"
path = "src/greeting.rs"
```

그리고 각 파일을 실행할 때:
```bash
cargo run --bin hello
cargo run --bin greeting
```

**학습 팁**: 처음에는 `src/main.rs` 하나만 사용하는 간단한 프로젝트로 시작하고, 필요에 따라 점진적으로 여러 파일로 확장하는 것을 추천합니다.

### 6.3 코드 분석

> **예제 코드**: [`02_function_analysis.rs`](../examples/ch01/src/02_function_analysis.rs)

```rust
fn main() {
// ^^^ 함수 정의 키워드와 이름
//      main은 프로그램의 진입점
}
```

**fn main()의 의미:**
- `fn`: 함수 정의 키워드 (function)
- `main`: 특별한 함수 이름 - 프로그램 실행 시작점
- `()`: 매개변수가 없음
- `{ }`: 함수 본체

```rust
println!("Hello, world!");
//^^^^^^^ 매크로 이름
//       ^^^^^^^^^^^^^^^ 문자열 리터럴
//                       ! 매크로 호출임을 나타냄
//                        ; 문장 종료
```

**println!의 특징:**
- 매크로입니다 (함수가 아님!)
- `!`가 매크로 호출임을 나타냄
- `;`으로 문장 종료

### 6.4 함수 vs 매크로

| 특성 | 함수 | 매크로 |
|------|------|--------|
| 호출 | `func()` | `macro!()` |
| 인자 개수 | 고정됨 | 가변적 |
| 용도 | 재사용 가능한 코드 | 메타프로그래밍 |

## 7. println! 매크로 상세 분석

### 7.1 기본 사용법

> **예제 코드**: [`03_println_basics.rs`](../examples/ch01/src/03_println_basics.rs)

```rust
fn main() {
    // 간단한 문자열 출력
    println!("Hello, world!");

    // 여러 줄 출력
    println!("Line 1");
    println!("Line 2");
}
```

### 7.2 문자열 보간 (String Interpolation)

> **예제 코드**: [`04_string_interpolation.rs`](../examples/ch01/src/04_string_interpolation.rs)

`{}` 플레이스홀더를 사용하여 변수 값을 출력할 수 있습니다.

```rust
fn main() {
    let name = "철수";
    let level = 5;

    // {} 플레이스홀더
    println!("용사: {}", name);
    println!("레벨: {}", level);

    // 여러 개의 플레이스홀더
    println!("{} 레벨 {}의 용사", name, level);

    // 인자 번호로 순서 지정
    println!("{1} 레벨 {0}의 용사", level, name);
    // 출력: 철수 레벨 5의 용사
}
```

### 7.3 포맷팅 옵션

> **예제 코드**: [`05_formatting_options.rs`](../examples/ch01/src/05_formatting_options.rs)

```rust
fn main() {
    let number = 42;

    // 다양한 진법
    println!("10진수: {}", number);      // 42
    println!("2진수: {:b}", number);     // 101010
    println!("8진수: {:o}", number);     // 52
    println!("16진수: {:x}", number);    // 2a
    println!("16진수(대문자): {:X}", number);  // 2A

    // 정렬과 패딩
    println!("오른쪽 정렬: |{:5}|", 12);     // |   12|
    println!("왼쪽 정렬: |{:<5}|", 12);      // |12   |
    println!("가운데 정렬: |{:^5}|", 12);     // | 12  |

    // 0으로 패딩
    println!("0으로 패딩: {:05}", 12);       // 00012
}
```

### 7.4 디버깅 포맷

> **예제 코드**: [`06_debug_format.rs`](../examples/ch01/src/06_debug_format.rs)

```rust
fn main() {
    // :? - Debug 포맷
    println!("디버그: {:?}", (42, "hello"));

    // :#? - 예쁜 Debug 포맷 (여러 줄)
    let v = vec![1, 2, 3];
    println!("벡터: {:#?}", v);
}
```

출력:
```
디버그: (42, "hello")
벡터: [
    1,
    2,
    3,
]
```

### 7.5 특수 문자

> **예제 코드**: [`07_special_characters.rs`](../examples/ch01/src/07_special_characters.rs)

```rust
fn main() {
    // 이스케이프 시퀀스
    println!("줄바꿈\n입니다");
    println!("탭\t입니다");
    println!("큰따옴표\"입니다");
    println!("백슬래시\\입니다");

    // 원시 문자열 (이스케이프 처리 안 함)
    println!(r"C:\Users\name");

    // 원시 문자열 (따옴표 포함)
    println!(r#"이것은 "따옴표" 포함"#);
}
```
