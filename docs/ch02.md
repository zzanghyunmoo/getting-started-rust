## 1. 변수와 불변성

### 1.1 변수 선언 기본

Rust에서 변수는 `let` 키워드로 선언합니다:

```rust
fn main() {
    // 플레이어 이름 변수
    let player_name = "철수";

    // 직업 변수
    let player_class = "전사";

    // 변수 값 출력
    println!("용사의 이름: {}", player_name);
    println!("직업: {}", player_class);
}
```

**변수 명명 규칙:**
- 변수 이름은 소문자로 시작
- 공백 대신 밑줄(_) 사용
- `snake_case` 스타일 권장
- 의미 있는 이름 사용

```rust
// ✅ 좋은 변수 이름
let player_hp = 100;
let max_mp = 50;
let is_alive = true;

// ❌ 피해야 할 이름
let a = 100;
let P = 50;
let x1 = true;
```

### 1.2 불변성의 기본

Rust의 변수는 **기본적으로 불변(immutable)**입니다:

```rust
fn main() {
    let player_name = "철수";
    println!("이름: {}", player_name);

    // player_name = "민수";  // 컴파일 에러!
    // error[E0384]: cannot assign twice to immutable variable `player_name`
}
```

**왜 기본적으로 불변일까요?**

1. **메모리 안전성**: 예기치 않은 변경 방지
2. **동시성 안전성**: 데이터 레이스 방지
3. **컴파일러 최적화**: 값이 변경되지 않음을 보장하여 공격적인 최적화 가능

```rust
// 불변 변수의 장점
fn main() {
    let max_hp = 100;
    let current_hp = 80;

    // max_hp는 실수로 변경될 위험 없음
    let hp_percent = (current_hp * 100) / max_hp;
    println!("HP: {}%", hp_percent);
}
```

### 1.3 가변 변수

`mut` 키워드로 가변 변수를 선언합니다:

```rust
fn main() {
    let mut player_hp = 100;
    println!("현재 HP: {}", player_hp);

    // 전투에서 피해를 입음
    player_hp = 85;
    println!("전투 후 HP: {}", player_hp);

    // 포션으로 회복
    player_hp = 100;
    println!("회복 후 HP: {}", player_hp);
}
```

**불변 변수 vs 가변 변수:**

| 특성 | 불변 변수 (`let`) | 가변 변수 (`let mut`) |
|------|-------------------|----------------------|
| 선언 | `let x = 5;` | `let mut x = 5;` |
| 값 변경 | 불가능 | 가능 |
| 대여 규칙 | 여러 불변 대여 가능 | 단일 가변 대여만 가능 |
| 기본값 | **기본** | 명시적 선언 필요 |

**언제 mut를 사용할까요?**

```rust
// ✅ mut가 적절한 경우: 상태 변경
fn main() {
    let mut hp = 100;
    hp -= 10;  // 전투 피해
    hp += 30;  // 회복
}

// ✅ 불변이 더 나은 경우: 계산 기반 값
fn main() {
    let max_hp = 100;
    let current_hp = 80;
    let hp_percent = (current_hp * 100) / max_hp;  // 재계산 필요 없음
}
```

### 1.4 변수의 메모리 표현

```rust
fn main() {
    let player_name = "철수";
    let player_hp = 100;
    let player_level = 1;
}
```

**메모리 레이아웃:**

```
┌─────────────────────────────────────┐
│         스택 메모리 (Stack)          │
├─────────────────────────────────────┤
│                                     │
│  [player_name]                      │
│   → "철수" (힙 데이터 참조)         │
│                                     │
│  [player_hp]                        │
│   → 100 (스택에 직접 저장)          │
│                                     │
│  [player_level]                     │
│   → 1 (스택에 직접 저장)            │
│                                     │
└─────────────────────────────────────┘
```

## 2. 상수 (Constants)

### 2.1 상수 선언 문법

```rust
// 게임 밸런스 상수
const MAX_HP: u32 = 100;
const MAX_MP: u32 = 50;
const STARTING_GOLD: u32 = 100;

fn main() {
    println!("최대 HP: {}", MAX_HP);
    println!("최대 MP: {}", MAX_MP);
    println!("초기 골드: {}G", STARTING_GOLD);
}
```

**상수의 필수 요소:**
1. `const` 키워드
2. **반드시 타입 어노테이션** 필수
3. 상수 이름은 **UPPER_SNAKE_CASE**
4. 컴파일 타임에 계산 가능한 값만 할당 가능

### 2.2 상수 vs 불변 변수

| 특성 | 상수 (`const`) | 불변 변수 (`let`) |
|------|----------------|-------------------|
| 타입 어노테이션 | **필수** | 선택적 (추론 가능) |
| 할당 시점 | 컴파일 타임 | 런타임 |
| 값 범위 | 상수 표현식만 | 모든 표현식 |
| 메모리 위치 | 코드 섹션 (inline) | 스택/데이터 섹션 |
| 스코프 | 전역 또는 블록 | 블록 |

```rust
// 상수 - 컴파일 타임에 결정
const MAX_HP: u32 = 100;

// 불변 변수 - 런타임에 결정 가능
let size = get_user_input();  // 함수 호출 결과

fn get_user_input() -> u32 {
    100
}
```

### 2.3 게임 밸런스 상수 예시

```rust
// 게임 밸런스 상수
const MAX_HP: u32 = 100;
const MAX_MP: u32 = 50;
const STARTING_GOLD: u32 = 100;
const LEVEL_UP_EXP: u32 = 100;

const WARRIOR_HP: u32 = 100;
const WARRIOR_MP: u32 = 50;
const WARRIOR_ATTACK: i32 = 15;
const WARRIOR_DEFENSE: i32 = 5;

const MAGE_HP: u32 = 60;
const MAGE_MP: u32 = 100;
const MAGE_ATTACK: i32 = 10;
const MAGE_DEFENSE: i32 = 3;

const ROGUE_HP: u32 = 80;
const ROGUE_MP: u32 = 60;
const ROGUE_ATTACK: i32 = 12;
const ROGUE_DEFENSE: i32 = 8;

fn main() {
    println!("=== 게임 밸런스 설정 ===");
    println!("최대 HP: {}", MAX_HP);
    println!("최대 MP: {}", MAX_MP);
    println!("초기 골드: {}G", STARTING_GOLD);
    println!("레벨업 경험치: {}", LEVEL_UP_EXP);

    println!("\n=== 직업별 스탯 ===");
    println!("전사: HP {} MP {} 공격력 {} 방어력 {}",
        WARRIOR_HP, WARRIOR_MP, WARRIOR_ATTACK, WARRIOR_DEFENSE);
    println!("마법사: HP {} MP {} 공격력 {} 방어력 {}",
        MAGE_HP, MAGE_MP, MAGE_ATTACK, MAGE_DEFENSE);
    println!("도적: HP {} MP {} 공격력 {} 방어력 {}",
        ROGUE_HP, ROGUE_MP, ROGUE_ATTACK, ROGUE_DEFENSE);
}
```

## 3. 섀도잉 (Shadowing)

### 3.1 섀도잉의 정의와 문법

섀도잉은 **이전 변수와 같은 이름의 새 변수를 선언**하는 것입니다:

```rust
fn main() {
    let x = 5;
    let x = x + 1;   // 섀도잉: x는 6
    let x = x * 2;   // 또 다른 섀도잉: x는 12

    println!("x의 최종 값: {}", x);  // 12
}
```

### 3.2 섀도잉 vs mut

| 특성 | 섀도잉 | `mut` |
|------|--------|-------|
| 문법 | `let x = x + 1;` | `x = x + 1;` |
| 값 변경 | 새 변수 생성 | 기존 변수 변경 |
| 타입 변경 | **가능** | 불가능 |
| 불변성 유지 | **가능** | 불가능 |

```rust
// 섀도잉 - 타입 변경 가능
fn main() {
    let spaces = "   ";           // &str 타입
    let spaces = spaces.len();    // usize 타입으로 변경
    println!("공백 개수: {}", spaces);
}

// mut - 타입 변경 불가능
fn main() {
    let mut spaces = "   ";
    // spaces = spaces.len();  // 컴파일 에러!
}
```

### 3.3 실전 사용 사례

**데이터 변환 파이프라인:**

```rust
fn main() {
    // 1단계: 입력 (공백 포함)
    let name = "  철수  ";

    // 2단계: 섀도잉 - 공백 제거
    let name = name.trim();

    // 3단계: 섀도잉 - 유효성 검사 후 기본값
    let name = if name.is_empty() {
        "모험가"
    } else {
        name
    };

    println!("용사의 이름: {}", name);
}
```

### 3.4 스코프 시각화

```rust
fn main() {
    let x = 5;           // 스코프 1: x는 5

    {
        let x = x * 2;   // 스코프 2: 내부 섀도잉, x는 10
        println!("내부 x: {}", x);  // 10

        {
            let x = x + 1;  // 스코프 3: 더 내부, x는 11
            println!("더 내부 x: {}", x);  // 11
        }
        // 스코프 3 종료
    }
    // 스코프 2 종료

    println!("외부 x: {}", x);  // 5
}
```

**스코프 다이어그램:**

```
스코프 레벨 0 (main 함수):
    let x = 5;           ━━ x는 여기서 5

    스코프 레벨 1:
        let x = x * 2;   ━━ 내부 섀도잉: x는 10
        println!("내부: {}", x);  → 10

        스코프 레벨 2:
            let x = x + 1; ━━ 더 내부: x는 11
            println!("더 내부: {}", x);  → 11

        println!("다시 내부: {}", x);  → 10

    println!("외부: {}", x);  → 5
```

## 4. 기본 데이터 타입

### 4.1 정수형 (Integer Types)

**부호 있는 정수형 (Signed Integers):**

| 타입 | 크기 | 범위 | 게임 사용 예시 |
|------|------|------|---------------|
| `i8` | 8-bit | -128 ~ 127 | 작은 보정값 |
| `i16` | 16-bit | -32,768 ~ 32,767 | 중간 크기 값 |
| `i32` | 32-bit | ±21억 | **공격력, 방어력** (기본값) |
| `i64` | 64-bit | 매우 큰 범위 | 큰 누적 값 |
| `i128` | 128-bit | 엄청 큰 범위 | 특수 목적 |
| `isize` | arch | CPU에 따름 | 인덱싱 |

**부호 없는 정수형 (Unsigned Integers):**

| 타입 | 크기 | 범위 | 게임 사용 예시 |
|------|------|------|---------------|
| `u8` | 8-bit | 0 ~ 255 | 아이템 개수, 슬롯 |
| `u16` | 16-bit | 0 ~ 65,535 | 작은 카운터 |
| `u32` | 32-bit | 0 ~ 42억 | **HP, MP, 골드** (기본값) |
| `u64` | 64-bit | 0 ~ 1.8×10¹⁹ | 누적 경험치 |
| `u128` | 128-bit | 0 ~ 3.4×10³⁸ | 특수 목적 |
| `usize` | arch | CPU에 따름 | 인덱싱 |

```rust
fn main() {
    // 플레이어 스탯 (음수 없음)
    let hp: u32 = 100;
    let mp: u32 = 50;
    let gold: u32 = 150;

    // 공격력 (음수 가능: 디버프)
    let attack: i32 = 15;
    let defense: i32 = 5;

    // 경험치 (매우 큰 값 가능)
    let exp: u64 = 1_234_567_890;

    println!("HP: {} | MP: {}", hp, mp);
    println!("공격력: {} | 방어력: {}", attack, defense);
    println!("골드: {}G | 경험치: {}", gold, exp);
}
```

### 4.2 부동소수점형 (Floating-Point Types)

| 타입 | 크기 | 정밀도 | 표준 |
|------|------|--------|------|
| `f32` | 32-bit | 단일 정밀도 | IEEE 754-2008 |
| `f64` | 64-bit | 배정밀도 (기본값) | IEEE 754-2008 |

```rust
fn main() {
    let damage = 15.5;      // f64로 추론
    let heal_amount: f32 = 25.0;

    println!("데미지: {}", damage);
    println!("회복량: {}", heal_amount);

    // 계산 예시
    let final_hp = 100.0 - damage + heal_amount;
    println!("최종 HP: {}", final_hp);
}
```

### 4.3 불리언형 (Boolean Type)

```rust
fn main() {
    let is_alive: bool = true;
    let is_poisoned: bool = false;
    let has_key: bool = true;

    // 논리 연산
    if is_alive && has_key {
        println!("문을 열 수 있습니다!");
    }

    if is_poisoned {
        println!("독 상태입니다!");
    }
}
```

**불리언 연산:**

| 연산자 | 설명 | 예시 |
|--------|------|------|
| `&&` | AND (둘 다 true면 true) | `true && false = false` |
| `\|\|` | OR (하나라도 true면 true) | `true \|\| false = true` |
| `!` | NOT (true/false 반전) | `!true = false` |

### 4.4 문자형 (Character Type)

```rust
fn main() {
    let first_char: char = '철';
    let emoji: char = '⚔️';
    let grade: char = 'A';

    println!("이름의 첫 글자: {}", first_char);
    println!("무기 이모지: {}", emoji);
    println!("등급: {}", grade);
}
```

**문자 vs 문자열:**

| 타입 | 설명 | 예시 |
|------|------|------|
| `char` | 4바이트, 1 문자 | `'A'`, `'가'`, `'😀'` |
| `&str` | 문자열 슬라이스 | `"Hello"` |

## 5. 타입 추론과 어노테이션

### 5.1 타입 추론

Rust는 컴파일러가 자동으로 타입을 추론합니다:

```rust
fn main() {
    // 컴파일러가 자동으로 타입 추론
    let hp = 100;        // i32로 추론
    let damage = 15.5;   // f64로 추론
    let is_alive = true; // bool로 추론

    println!("HP: {}", hp);
    println!("데미지: {}", damage);
    println!("생존: {}", is_alive);
}
```

### 5.2 타입 어노테이션

**필수적인 경우:**

```rust
// 1. 모호한 경우
fn main() {
    let guess: u32 = "42".parse().expect("숫자가 아닙니다!");
    // .parse()는 제네릭 → 타입 명시 필요
    println!("추측: {}", guess);
}

// 2. 다른 타입으로 지정
fn main() {
    let hp: u32 = 100;  // u32로 명시
    let max_hp: i64 = 100;  // i64로 명시
}
```

### 5.3 추론의 한계

```rust
// 1. 제네릭 메서드
fn main() {
    let x = "42".parse();
    // 에러: type annotations needed

    let x: i32 = "42".parse().unwrap();  // 해결
}

// 2. 빈 컬렉션
fn main() {
    let vec = Vec::new();
    // 에러: type annotations needed

    let mut vec: Vec<i32> = Vec::new();  // 해결
}
```

## 6. 타입 변환

### 6.1 명시적 타입 변환

Rust는 암시적 타입 변환을 제공하지 않습니다. `as` 키워드를 사용합니다:

```rust
fn main() {
    let hp_float: f64 = 100.0;
    let hp_int: u32 = 100;

    // 명시적 변환
    let hp_total = hp_int + (hp_float as u32);

    // 안전한 변환
    let damage: i32 = -15;
    let abs_damage = damage.abs() as u32;

    println!("총 HP: {}", hp_total);
    println!("절대값 데미지: {}", abs_damage);
}
```

### 6.2 변환 호환성

| 원본 타입 | as u32 | as i32 | as f64 | as char |
|-----------|--------|--------|--------|---------|
| i32 | ⚠️ | ✅ | ✅ | ❌ |
| u32 | ✅ | ⚠️ | ✅ | ⚠️ |
| f64 | ⚠️ | ⚠️ | ✅ | ❌ |

- ✅: 전체 범위 변환 가능
- ⚠️: 범위 초과 시 잘림
- ❌: 불가능

## 7. 복합 타입 기초

### 7.1 튜플 (Tuple)

**선언과 사용:**

```rust
fn main() {
    // 위치 좌표
    let position: (i32, i32) = (10, 20);

    // 해체 (destructuring)
    let (x, y) = position;

    println!("위치: ({}, {})", x, y);

    // 인덱스로 접근
    println!("X 좌표: {}", position.0);
    println!("Y 좌표: {}", position.1);
}
```

### 7.2 배열 (Array)

**선언과 사용:**

```rust
fn main() {
    // 직업별 능력치
    let warrior_stats: [i32; 4] = [100, 50, 15, 5];  // HP, MP, 공격, 방어
    let mage_stats: [i32; 4] = [60, 100, 10, 3];
    let rogue_stats: [i32; 4] = [80, 60, 12, 8];

    println!("전사 HP: {}", warrior_stats[0]);
    println!("마법사 MP: {}", mage_stats[1]);
    println!("도적 공격력: {}", rogue_stats[2]);

    // 같은 값으로 초기화
    let inventory = [0; 20];  // 빈 인벤토리 20칸
}
```

### 7.3 유닛 타입 `()`

```rust
fn main() {
    let unit: () = ();
    // 길이가 0인 튜플
    // 크기: 0 bytes

    // 용도: 반환값이 없는 함수
}

fn nothing() -> () {
    // 아무것도 반환 안 함
}
```

### 8.2 단계별 구현

**[main.rs](dungeon_crawler/src/main.rs)를 다음과 같이 수정하세요:**

```rust
//! Dungeon Crawler - 캐릭터 생성 시스템

/// 게임 밸런스 상수
const MAX_HP: u32 = 100;
const MAX_MP: u32 = 50;
const STARTING_GOLD: u32 = 100;

/// 캐릭터 생성 시스템
fn main() {
    // 1. 상수 출력
    println!("=== 게임 밸런스 ===");
    println!("최대 HP: {}", MAX_HP);
    println!("최대 MP: {}", MAX_MP);
    println!("초기 골드: {}G", STARTING_GOLD);
    println!();

    // 2. 플레이어 이름 (불변)
    let player_name = "철수";

    // 3. 직업 (불변)
    let player_class = "전사";

    // 4. 직업별 초기 능력치
    let (hp, mp, attack, defense) = match player_class {
        "전사" => (100, 50, 15, 5),
        "마법사" => (60, 100, 10, 3),
        "도적" => (80, 60, 12, 8),
        _ => (80, 50, 10, 5),  // 기본값
    };

    // 5. 캐릭터 정보 출력
    println!("====================================");
    println!("캐릭터가 생성되었습니다!");
    println!("====================================");
    println!("이름: {}", player_name);
    println!("직업: {}", player_class);
    println!("HP: {} / MP: {}", hp, mp);
    println!("공격력: {} / 방어력: {}", attack, defense);
    println!("골드: {}G", STARTING_GOLD);
    println!("====================================");
}
```

## 요약

### 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **변수** | `let` 키워드로 선언, 기본적으로 불변 |
| **가변 변수** | `let mut`로 선언, 값 변경 가능 |
| **상수** | `const`로 선언, 컴파일 타임에 결정 |
| **섀도잉** | 같은 이름으로 새 변수 선언, 타입 변경 가능 |
| **정수형** | i8~i128, u8~u128 (HP, MP, 골드 등) |
| **부동소수점** | f32, f64 (데미지 계산 등) |
| **불리언** | bool (생존 여부, 키 보유 등) |
| **문자** | char (4바이트 Unicode) |
| **튜플** | 서로 다른 타입의 값 묶음 |
| **배열** | 같은 타입의 값 고정 길이 묶음 |
